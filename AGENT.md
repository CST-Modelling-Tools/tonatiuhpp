# Agent Working Rules

- Optimize for minimal token consumption.
- Work in milestone mode, not micro-step mode.
- Prefer the fewest meaningful commits that preserve clarity.
- Do not spend time on cosmetic refactors, renames, or wording cleanups unless they affect correctness, safety, maintainability, or release readiness.
- Keep progress updates brief and infrequent.
- Keep final summaries concise.
- When a feature is mostly complete, stop making small improvements and switch to a completion review.
- Only propose further edits if they materially improve correctness, security, cross-platform behavior, CI stability, or release readiness.
- Batch related changes together instead of splitting into many tiny steps.
- Ask for confirmation only when there is a real tradeoff or product decision.
